使用Zmap及脚本检测现有GoogleIP
发布于：2018-06-03 18:55:09 标签：/ github / linux / 访问：

自从收到来自Censys的邮件通知称其将开始收费计划以后，免费用户的API查询次数就被限制在250次/月，这使得给我的IP更新带来了很大不便……特别像更新US、RU这种IP资源非常大的国家，很快次数就会被用完（关键它的站点现在还限制免费用户使用search API只能查询前10000个结果，后面要查询还得花钱）。因此，我下定决心开始自己搜集IP！（认真脸

方法很简单：

  1. 使用Zmap对指定国家的IP段进行端口扫描（我使用的是443端口）；
  2. 待扫描完成后，提取结果，使用checkip.py进行证书扫描
  3. 对结果排序，去重并修改为CIDR格式即可

选择Zmap进行端口扫描是因为它耗时短且准确率高（相较Nmap而言，Masscan也不错，不过它不能直接输出IP），实测扫完越南所有IPv4段在我的VPS上用了20min左右（限制带宽10M且只占用一半CPU）。最后又花了半个小时用脚本检测了得到的33万多结果，并最终完成了更新。

看来以后更新得靠这办法了……虽然慢，但也很靠谱（只能说Censys坑爹）XD
