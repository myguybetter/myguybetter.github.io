误删除Linux系统中ssl证书文件夹后的修复操作
发布于：2018-05-30 15:53:12 标签：/ linux / 访问：

由于自己一时脑抽，一不小心把VPS上的/etc/ssl文件夹直接给删了……结果每次从服务器上curl或者git clone一些https链接的资源就报证书错误:(

可是又不想重装系统，于是就请教Google看能否补救一下，搜出来的都是“安装ca-certificates包”，结果输入命令发现是安装好的，怎么办呢？

解决方案：先新建/etc/ssl及/etc/ssl/certs文件夹，之后直接运行update-ca-certificates命令即可……对，就是这么简单XD
